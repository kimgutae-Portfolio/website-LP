# 지역 사업자용 홈페이지 서비스 LP — 제작 사양서

> 이 문서는 사업 기획 논의를 정리한 것. 코딩 에이전트는 이 사양대로 LP를 구현할 것.
> 대상 언어: 사이트 콘텐츠는 **일본어**, 코드 주석 등은 자유.

---

## 1. 배경 / 사업 모델

- 운영자: 오사카(堺市) 거주 웹 엔지니어. 개인사업자. 기존 포트폴리오 사이트 gutae.dev (시스템 개발 본진) 별도 운영 중.
- 본명 상품은 **웹 시스템 개발** (예약, LINE 미니앱, 엑셀 업무 자동화 등).
- 홈페이지 월정액은 **입구 상품**: 지역 사업자와 월액 계약 관계를 만들고, 매달 접점(상담)을 통해 시스템 안건을 발굴 → 견적 경쟁 없이 수주하는 구조.
- 따라서 이 LP는 "제작 작품 어필"이 아니라 **"가격이 붙은 기성품 카탈로그 + 실물 견본"** 역할.
- 이 LP 자체가 고객에게 팔 **1페이지 템플릿의 1호 적용 사례**가 되도록 만든다.
  (영업 시 "이 페이지가 월 5,500엔 상품의 실물입니다"라고 보여주는 용도)

## 2. 타겟

- 堺市 및 인접 지역의 소규모 점포·사업소 (음식점, 미용실, 정골원, 치과, 개호 시설 등 1~10인 규모)
- 특징: 사이트가 없거나 낡음 / 만든 업자와 연락 두절 / 스마트폰 대응 안 됨 / IT 담당자 부재
- 열람 환경: **스마트폰 우선** (모바일 퍼스트 필수)

## 3. 상품 / 요금 (LP에 게재할 내용)

### 메인 플랜
- **월액 5,500엔(税込) / 初期費用 0円** — 「開業記念キャンペーン価格」로 표기 (통상가는 미표기. 존재하지 않는 통상가와의 비교 표시는 二重価格表示 문제가 되므로 금지)
- 내용: 1페이지형 홈페이지 제작 + 서버·도메인 관리 + SSL + 갱신 연 4회(텍스트·사진 교체, 1회 3항목까지) + **LINE 친구추가 버튼(공식계정 연결 아이콘) 기본 세팅**
- 최저 계약기간: **12개월** (이후 언제든 해지 가능)
- 디자인: 3종 템플릿에서 선택 (완전 커스텀 없음 — 제작 공수 반나절~1일로 압축하는 전제)

### 옵션 (간단히 언급만)
- LINE 공식계정 구축 세트: +5,500엔/월
- 다페이지·시스템 연동: 별도 견적 (상위 플랜은 추후 추가 예정이므로 확장 여지를 둔 레이아웃)

### 요금 관련 규칙
- 가격 표기는 税込 총액 표시: 「月額5,500円（税込）」
- FAQ에 명기: 도메인은 고객 소유 / 해지 시 사이트 정지, 데이터 인도는 유상
- 갱신 범위를 숫자로 명시 (범위 무한정 방지)

## 4. LP 구성 (1페이지 스크롤, 위에서부터)

블로그 없음. 추가 페이지 없음. 아래 섹션만.

### 4-1. 히어로
- 카피(안): 「初期費用0円・月額5,500円。堺の店舗・事業所のホームページ、作って終わりにしません。」
- 서브: 「ホームページ制作から、予約・LINE・業務システムまで。堺の事業者の"IT担当"になります。」
  (※ 시스템도 하는 사람임을 처음부터 인지시켜 업셀 복선을 심는 문구 — 반드시 포함)
- CTA 버튼: 「LINEで気軽に相談」 (주요 CTA는 전 섹션 공통으로 LINE)

### 4-2. こんなお悩みありませんか (3~4개)
- 「ホームページが古いまま、スマホで見ると崩れる」
- 「作った業者と連絡が取れない・更新を頼めない」
- 「グーグルで検索しても自分のお店が出てこない」
- 「毎月の手作業（エクセル・予約管理）をなくしたい」 ← 시스템 니즈 필터용

### 4-3. 制作イメージ (데모 링크 2~3개)
- 가상 점포 데모 사이트 링크 카드. 초기엔 「準備中」 표시 허용, 자리만 확보.
- 하단에 「※架空の店舗のデモサイトです」 표기.

### 4-4. 料金
- 위 3장의 플랜을 표/카드로. 포함 범위(갱신 횟수 등) 명기.
- 「開業記念キャンペーン」 배지를 켜고 끌 수 있게 구현 (config로 토글). 기간·조건 문구도 config에서 수정 가능하게.

### 4-5. 制作の流れ (3스텝)
- ①ヒアリング（堺市内は直接お伺いします） → ②デザイン選択・原稿確認 → ③公開（最短1週間）
- 「直接お伺いします」는 지역 밀착 차별화 포인트 — 강조.

### 4-6. 운영자 소개
- 얼굴 사진 자리(placeholder) + 屋号 + 이름 + 「堺市在住のWebエンジニア」 + 한마디.
- 「システム開発の実績はこちら」 → gutae.dev 링크.

### 4-7. FAQ (4~6개)
- 最低契約期間は？ → 12ヶ月、以降いつでも解約可
- 解約したらサイトは？ → 公開停止。データお引き渡しは有償対応
- ドメインは誰のもの？ → お客様の所有です（取得代行可）
- 写真や文章がなくても大丈夫？ → ヒアリングで一緒に作ります
- 今あるサイトからの移行は？ → 可能。無料で現状診断します

### 4-8. CTA (최하단)
- LINE 친구추가 버튼(QR 포함) + 문의 폼 (이름 / 상호 / 연락처 / 상담 내용)
- 폼 전송은 추후 Supabase 저장 + 알림 연동 예정. 초기엔 API 라우트 stub 또는 메일 전송으로 OK.

### 공통
- **플로팅 LINE 친구추가 아이콘**: 화면 우하단 고정, 전 섹션에서 항상 표시 (LINE 공식계정 친구추가 링크로 연결). 이 아이콘은 고객 사이트 템플릿에도 기본 탑재되는 사양이므로 컴포넌트로 분리.
- 푸터: 「運営者: ◯◯（屋号）」 / 特定商取引法に基づく表記 페이지 링크(별도 1페이지, 내용은 placeholder 가능) / gutae.dev 링크

## 5. 디자인 방향

- 콘셉트: 「호화」가 아니라 **깨끗·빠름·스마트폰에서 완벽**. 로드 속도와 모바일 완성도 자체가 세일즈 포인트.
- 5,500엔 상품의 견본이므로 과한 장식·애니메이션 금지. 신뢰감 있는 로컬 비즈니스 톤.
- 글자 크게, 여백 넉넉히 (열람자는 40~60대 사업주 상정).
- 일본어 폰트, 일본 로컬 상거래 관행에 맞는 표기 (税込 표기 등).
- Lighthouse 모바일 성능 90+ 목표.

## 6. 기술 스택

- **Next.js (App Router, 정적 생성 위주) + Vercel 배포**
- 스타일: Tailwind CSS
- 폼: 우선 Next API Route → 메일 전송(Resend) 또는 console stub. 추후 Supabase 저장 + LINE 알림으로 확장.
- 구조: 나중에 이 코드를 **고객용 템플릿**으로 재사용할 것이므로,
  - 사이트 고유 정보(상호, 가격, 카피, 색, 데모 링크, 모니터가격 토글)는 **config 파일 1곳**(예: `site.config.ts`)로 분리
  - 섹션은 컴포넌트 단위로 분리 (Hero, Pain, Demo, Pricing, Flow, Owner, Faq, Cta)
- 다국어 불필요 (일본어만)

→ **당사 5,500엔은 로컬 월액형 최저가권.** 가격 자체는 시세 안이므로 가격을 주역으로 만들지 말 것.

### 경쟁사 계약 조건에서 발견한 차별화 포인트 (LP FAQ에서 대비가 드러나게 쓸 것)
- 경쟁 저가 플랜의 흔한 조건: 독자 도메인 미제공(자사 도메인 하위 URL) / 해지 시 성과물 전부 파기 / 사진 촬영 없음
- 당사 조건: **도메인은 고객 소유** / **해지 시 데이터 유상 인도(파기 아님)** / 히어링 방문으로 원고를 함께 제작
- 소구 우선순위: ①堺 거주·직접 방문 ②도메인·데이터의 양심 조건 ③시스템까지 확장 가능 — 가격은 그 다음.

## 8. 인프라 / 운영 메모 (참고용, LP 코드 범위 외)

- 도메인: 신규 취득 예정 (.jp 우선, 일본계 레지스트라 / .com이면 Cloudflare). gutae.dev와 분리.
- DNS: Cloudflare. 메일 수신은 Cloudflare Email Routing → Gmail 포워딩, 발신은 Resend SMTP (SPF/DKIM/DMARC 설정).
- Vercel은 상용 이용이므로 Pro 플랜 (계정당 과금, 프로젝트 다수 수용).
- 향후 시스템 상품: Supabase(도쿄, 멀티테넌트+RLS) / Python 백엔드가 필요한 경우 Render 워커 (엑셀·OCR·배치 담당).

## 9. 완료 기준 (Definition of Done)

1. 위 8개 섹션이 구현된 1페이지 LP (일본어 카피 반영)
2. 모바일 퍼스트, 반응형 완성
3. config 파일 수정만으로 상호·가격·문구·색이 교체 가능 (템플릿화 검증)
4. 문의 폼 동작 (stub 가능)
5. Vercel 배포 가능한 상태 (빌드 에러 없음)
6. 特商法 표기 페이지(placeholder) 및 푸터 링크

## 10. 이후 로드맵 (참고)

1. LP 공개 → 도메인·메일 연결
2. 데모 사이트 1호(개호 시설) 제작 → LP의 制作イメージ에 연결
3. 영업 시작: 기존 개호 고객사 소개 부탁 → 근처 방문 / 목업 첨부 메일 (특정전자메일법 준수: 발신자 표기 + 수신거부 안내)
4. 3곳 모니터 계약 확보 → 후기·사례 게재
5. 월 1회 상담에서 시스템 니즈 발굴 → 미니 시스템(10만엔~ + 월액) 업셀
